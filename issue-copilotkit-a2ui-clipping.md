Title
A2UI surfaces clip bottom buttons in CopilotChat (overflow hidden in A2UIMessageRenderer SurfaceHost)

## Description
When an A2UI surface is “tall” (e.g. a `Card` containing a bottom `Row` of `Button`s), the bottom actions get visually clipped in the chat UI (you only see the top of the buttons, not the label text). This happens even though the overall chat may be scrollable.

From debugging locally, the clipping appears to be caused by the A2UI activity renderer’s SurfaceHost applying `overflow: hidden` (and `height: 100%`) to the `<themed-a2ui-surface>` host element, and also wrapping it in an `overflow-hidden` container.

## Affected package / version
- `@copilotkit/a2ui-renderer` `1.51.3`

## Environment
- Next.js app embedding `CopilotChat`
- A2UI rendered via `renderActivityMessages` with activity type `"a2ui-surface"`

## Repro steps
1. Render an A2UI surface in the chat with enough vertical content to require scrolling.
2. Include a bottom actions row with one or more Buttons (e.g. “Add to calendar” / “Discard”).
3. Observe the bottom buttons are clipped (text not visible).

## Expected behavior
- Bottom action buttons should remain visible.
- Long A2UI surfaces should either:
  - expand naturally and rely on the parent scroll container, or
  - become scrollable themselves (recommended), but not clip content.

## Actual behavior
- Bottom actions (buttons) are clipped; only the top portion of the button is visible.

## Root cause (suspected)
In the current renderer, SurfaceHost renders something equivalent to:
- Wrapper: `overflow-hidden`
- Surface host element: inline `style={{ height: "100%", overflow: "hidden" }}`

These two constraints can clip content for tall surfaces.

## Minimal patch that fixed it locally
Patched file in the installed npm package:
- `node_modules/@copilotkit/a2ui-renderer/dist/A2UIMessageRenderer.js`

Location:
- `function SurfaceHost(...) { ... return (...) }`

Changes:
1) Wrapper div className
- Before:
  ```js
  className: "flex w-full flex-none overflow-hidden rounded-lg bg-white/5 p-4"
  ```
- After:
  ```js
  className: "flex w-full flex-none overflow-visible rounded-lg bg-white/5 p-4"
  ```

2) `<themed-a2ui-surface>` inline style
- Before:
  ```js
  style: { height: "100%", overflow: "hidden" }
  ```
- After:
  ```js
  style: { height: "auto", maxHeight: "70vh", overflow: "auto", display: "block" }
  ```

Result after patch:
- A2UI surfaces no longer clip bottom buttons; long surfaces become scrollable within the message.
